# ホームページ運用開始ガイド

AI Compass Journalを、外部サービス接続前でも「生きているサイト」として使うための運用メモです。

## 毎日やること

- noteを書く
- 反応が良かったnoteを次の更新候補にする
- `/updates` に追加する候補をメモする
- `/free` へ戻すCTAをnote末尾に入れる

## 週1でやること

- `/library` に載せる記事候補を整理する
- `/products` の公開中、近日公開、準備中ステータスを確認する
- `/free` の説明文や読まれる順番を改善する
- クリックされている導線を確認する
- 未接続サービスを稼働中に見せていないか確認する

## 月1でやること

- 無料スターターキットを改善する
- 有料商品候補を整理する
- legal正式化の進捗を確認する
- BOOTH、メルマガ、コミュニティ接続を検討する
- `/updates` を見直し、古い表現を整理する

## 外部サービス接続前の運用ルール

- 未接続を稼働中に見せない
- 準備中リンクはクリック不能、または「準備中」と明示する
- サイト内で価値が完結する導線を優先する
- 無料スターターキットは `/free` と `public/free-starter-kit/` を正とする
- 外部URLが確定したら `src/config/site.ts` と `src/data/*` を更新する
- 秘密情報、APIキー、個人情報はリポジトリに入れない

## 更新情報の追加ルール

`src/data/updates.ts` に1件追加します。

- `id`: 英数字とハイフンで一意にする
- `date`: `YYYY-MM-DD`
- `title`: 読者に見せる短い更新名
- `description`: 何が使えるようになったかを書く
- `href`: 関連ページ
- `category`: 公開、無料キット、導線、SEOなど

追加後は `npm run lint`, `npm run typecheck`, `npm run build` を実行します。
