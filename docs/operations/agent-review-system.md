# Agent Review System

## 目的

AI Compass Journal の更新を、単一エージェントの場当たり実行ではなく、役割分担されたレビュー体制で安定運用するための文書です。

## エージェント役割

### Codex

- 実装
- ローカル検証
- commit作成
- 変更差分の整理
- fresh clone ベースの安全な作業

Codex は、コード編集と実行確認の担当です。まずローカルで再現し、差分を最小化し、どこまで通ってどこで止まったかを明確に残します。

### ChatGPT

- 戦略
- 安全境界
- 最終レビュー
- 優先順位整理
- エージェント間の役割整理

ChatGPT は、実装そのものよりも、方向性と安全境界を揃える役割を持ちます。

### Claude

- 文章品質
- 読みやすさ
- 編集レビュー
- 冗長表現や曖昧表現の是正

Claude は、記事や文書の自然さ、読者負荷、構成の滑らかさを見る役割です。

### SEO reviewer

- title / description
- 検索意図
- 内部リンク
- Search Console 観点
- index / sitemap 観点

SEO reviewer は、検索流入とページ設計の整合性を見る補助レビュアーです。

### Human owner

- 最終マージ判断
- 公開境界の判断
- 未確認情報の最終確認
- 外部公開前の責任ある承認

最終責任は常に Human owner が持ちます。

## PRごとのレビュー観点

PR は最低でも次の5観点で見る前提にします。

### 技術レビュー

- 変更ファイルは適切か
- 既存構造を壊していないか
- lint / typecheck / build の結果は妥当か
- 失敗時に今回差分起因か既存要因か切り分けられているか

### コンテンツレビュー

- note転載になっていないか
- 読者が再現できる粒度か
- 過剰な断定がないか
- プロンプト、例、注意点の粒度が実務向けか

### SEOレビュー

- title が検索意図と一致しているか
- description が本文とズレていないか
- 内部リンクが自然か
- index 対象として公開価値があるか

### 安全境界レビュー

- API key や秘密情報が入っていないか
- note 投稿系に誤って触っていないか
- 触ってはいけない data / workflow / 実験領域に触れていないか
- 未確認情報の断定がないか

### 公開後確認

- 本番で表示崩れがないか
- CTA が成立しているか
- リンク切れがないか
- 公開した価値が `/updates` や他ページ導線に反映されているか

## 完全自動化しない工程

次の工程は、今後も完全自動化しない前提で扱います。

- マージ判断
- 未確認情報の最終確認
- note 本文転載の確認
- 個人情報・秘密情報の確認

これらは品質だけでなく責任境界の問題でもあるため、人間判断を残します。

## 推奨フロー

1. Codex が fresh clone で実装・検証・commit を作成する
2. ChatGPT が安全境界と方向性を確認する
3. Claude が文章・可読性を確認する
4. SEO reviewer が検索意図と title / description を確認する
5. Human owner が merge 可否を判断する

## 重要原則

- 1つのエージェントで全部完結させない
- 実装レビューと公開判断を分ける
- 「通ったか」だけでなく「何が未解決か」も残す
- 自動化率を上げても、責任境界は曖昧にしない
