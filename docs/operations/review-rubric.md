# Review Rubric

## 目的

AI Compass Journal の PR を、技術・コンテンツ・SEO の観点で継続的にレビューできるようにするための共通基準です。

## use case 詳細ページの評価基準

### 再現可能性

- 読者が何をすればよいか順番に分かるか
- 手順が抽象論で終わっていないか
- 1回試せる粒度になっているか

### プロンプト品質

- そのままコピペして試せるか
- 変数や条件が分かるか
- AI に任せてはいけない点が含まれているか

### 入力例・出力例

- 現実的な例か
- 入力だけ、または出力だけで終わっていないか
- 完成イメージが読者に伝わるか

### 注意点

- チェックポイントがあるか
- よくある失敗が具体的か
- 改善プロンプトまで含まれているか

### note転載回避

- note 本文の丸写しになっていないか
- HP 用の再現手順として再構成されているか

### 未確認情報の断定回避

- 実測していない数値を断定していないか
- 外部仕様や権限条件を推測で書いていないか

## experiment 記事の評価基準

### 仮説

- 何を検証したかったかが明確か

### 方法

- 手順、条件、使用環境が分かるか

### 実測値

- 何をどう測ったかが分かるか
- 主観だけで終わっていないか

### 失敗ケース

- うまくいかなかった点が残っているか
- 失敗を隠していないか

### 自動化率定義

- 自動化率の意味が明確か
- どこまでを自動、どこからを人間判断としたか分かるか

### 考察

- 単なる結果報告で終わらず、原因や制約に触れているか

### 次に試すこと

- 次の実験に接続しているか
- 改善余地が具体的か

## SEO評価基準

### title

- 検索意図に合っているか
- 自然な日本語か

### description

- 本文とズレていないか
- 約束しすぎていないか

### 検索意図

- 読者が知りたいことと本文内容が一致しているか

### 内部リンク

- `/free` `/prompts` `/workflows` `/guides` `/ai-use-cases` などへの導線があるか

### sitemap / index 対象

- 検索流入を取るページとして公開価値があるか
- 逆に下書き・薄いページを誤って index させる構成になっていないか

## 運用メモ

- use case は「再現手順」
- experiment は「検証記録」
- SEO は「検索意図と導線」

この3軸を混ぜずに見ることが重要です。
