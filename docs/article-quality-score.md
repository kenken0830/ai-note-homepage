# 記事品質スコア

HP記事は100点満点で評価します。80点未満の記事は公開しません。80点未満の場合は、`articleBacklog` の候補や下書きに戻し、次の改善PRで再評価します。

## スコア表

| 項目 | 点数 | 見ること |
| --- | ---: | --- |
| 読者のやりたいことが明確 | 15 | 誰が何をしたいページか、タイトルと冒頭で分かる |
| 手順が具体的 | 15 | 読者が順番どおりに試せる粒度になっている |
| コピペ用プロンプトが使える | 15 | 素材、条件、出力形式、確認事項が入っている |
| 入力例・出力例がある | 10 | 読者が完成イメージを持てる |
| よくある失敗と改善策がある | 10 | 失敗例だけでなく直し方がある |
| 内部リンクがある | 10 | `/free` `/prompts` `/workflows` `/guides` `/ai-use-cases` へつながる |
| noteとの住み分けがある | 10 | noteは実験ログ、HPは完成版手順として分かれている |
| 未確認情報を断定していない | 10 | 統計、効果、外部サービス状態を推測で書いていない |
| title/descriptionが自然 | 5 | 日本語として自然で、検索意図と本文が一致する |

## 判定

- 90点以上: 公開してよい。必要ならSEO/UXだけ微調整する
- 80から89点: 公開可能。ただし弱い項目をPR本文に残す
- 79点以下: 公開しない。下書きまたはバックログに戻す

## 公開前チェック

- 読者の作業が1つに絞れている
- AIに渡す前に準備するものが明記されている
- プロンプトがそのまま使える
- 入力例と出力例が現実的
- 出力後に人間が確認するポイントがある
- よくある失敗が具体的
- 改善プロンプトがある
- 関連ページへのCTAがある
- `/updates` に記録している
- `npm run lint` `npm run typecheck` `npm run build` が通っている
- `next-env.d.ts` など生成差分が入っていない

## 減点例

- 読者が何をできるようになるか曖昧: -10
- 手順が3ステップ未満で抽象的: -10
- プロンプトが「整理してください」だけ: -10
- 入力例か出力例がない: -10
- AIの推測をそのまま採用させる: -10
- 外部サービスが未接続なのに登録・購入できるように見せる: -15
- note本文の転載に近い: -20
- 秘密情報、APIキー、個人情報が混入: P1、公開不可
