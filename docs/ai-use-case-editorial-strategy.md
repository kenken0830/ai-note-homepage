# AI活用ユースケース辞典 編集方針

このドキュメントは、noteの毎日投稿とホームページの `/ai-use-cases` をつなぎ、AI Compass Journalを「やりたいことから探す、AI活用の実践辞典」として育てるための編集方針です。

## 役割分担

note自動調査はネタ発掘装置です。日々のテーマ候補、読者が困っていそうな作業、試したいプロンプト、検証したいAI活用を見つけるために使います。

noteは毎日の調査・実験ログです。完成版にしようとせず、「何を試したか」「どこで詰まったか」「AIの出力をどう直したか」「次に何を試すか」を軽く残します。

ホームページは完成版のAI活用辞典です。note本文を丸ごと転載せず、再現できる手順、コピペ用プロンプト、確認ポイント、よくある失敗、関連ページへの内部リンクとして再編集します。

## `/ai-use-cases` に昇格する基準

noteで扱ったテーマは、次の条件を満たしたら `/ai-use-cases` に昇格します。

- 読者の反応、保存、質問、コメントがある
- 「会議メモを議事録にする」のように、やりたいことが明確で検索されやすい
- 手順、プロンプト、確認ポイント、失敗例まで書ける
- `/free` `/prompts` `/workflows` `/guides` のいずれかへ自然につながる
- ツール名の紹介ではなく、作業の再現手順として説明できる

AIニュースやツールまとめは、辞典の主役にしません。必要な場合も「そのツールで何を実現できるか」に変換します。

## 毎日追加するなら短いユースケースカード

毎日増やす場合は、完成版ページを急がず、まずは `/ai-use-cases` のカード候補として扱います。

- タイトル
- カテゴリ
- 何ができるか
- 想定タグ
- noteで試す角度
- 詳細ステータス

詳細ページが未完成のものは「詳細準備中」として扱い、本物リンクのように見せません。

## 週1で完成版ガイドに昇格する流れ

週1回、直近のnoteとカード候補を見直します。

1. 反応が良かったテーマを1つ選ぶ
2. 手順を3〜5ステップに整理する
3. コピペ用プロンプトを1つ作る
4. 出力を確認するポイントを3つ以上書く
5. よくある失敗を3つ以上書く
6. noteで書くならどういう実験ログになるかを追記する
7. `/free` `/prompts` `/workflows` `/guides` へ内部リンクを置く

この状態まで整ったものだけ、詳細ページとして公開します。

## `/updates` に記録するルール

次の変更をしたときは `/updates` に記録します。

- `/ai-use-cases` に新しいカテゴリや詳細ページを追加した
- 準備中カードを詳細ページへ昇格した
- 無料キットやプロンプト集と連動する内容を更新した
- noteで反応が良かったテーマをホームページへ整理した

更新情報は読者向けでもあり、運営者が育成履歴を追うための記録でもあります。

## 内部リンク方針

`/ai-use-cases` はやりたいこと別の入口です。各詳細ページから、次の役割で内部リンクを置きます。

- `/free`: 最初に使う無料キット
- `/prompts`: コピペ用プロンプト集
- `/workflows`: 複数ステップの流れ
- `/guides`: AI活用の考え方
- `/updates`: 追加・改善の記録

読者が「次に何をすればいいか」で迷わないように、外部サービス未接続の間はサイト内で完結する導線を優先します。
