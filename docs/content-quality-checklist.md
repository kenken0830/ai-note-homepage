# HP記事品質チェックリスト

AI Compass JournalのHP記事は、noteの転載ではなく、再現できるAI活用手順として公開します。

## 各HP記事に必ず入れる項目

- 何ができるか
- どんな人向けか
- 使う場面
- AIに渡す前に準備するもの
- 手順
- コピペ用プロンプト
- 入力例
- 出力例
- 確認ポイント
- よくある失敗
- 改善プロンプト
- 関連ページ
- `/free` へのCTA

## 書き方の基準

- 読者の「やりたいこと」をタイトルに入れる
- AIニュースやツール比較を主役にしない
- 手順は、はじめて読む人がそのまま試せる粒度にする
- プロンプトは素材を差し替えれば使える形にする
- 出力例は、架空の実績や未確認事実を含めない
- 最終判断、事実確認、固有情報の確認は人間が行うと明記する

## 公開前チェック

- 未確認情報を断定していないか
- note記事の丸ごと転載になっていないか
- 準備中サービスを稼働中に見せていないか
- 内部リンクがあるか
- `/free` へのCTAがあるか
- `/updates` に記録したか
- `npm run lint` `npm run typecheck` `npm run build` が通っているか

## 薄い記事の判定

次の状態なら公開せず、`src/data/articleBacklog.ts` の候補に留めます。

- 手順が2ステップ以下
- プロンプトが抽象的で、何を入れればよいか分からない
- 入力例または出力例がない
- よくある失敗が書けない
- 関連ページへ自然につながらない
- 「便利です」「効率化できます」だけで具体的な作業がない

## 改善時の見直し順

1. タイトルが検索意図と合っているか
2. 冒頭で何ができるか分かるか
3. 手順が再現できるか
4. プロンプトに入力条件と出力形式があるか
5. 確認ポイントが実務で使えるか
6. `/free` `/prompts` `/workflows` `/guides` へつながっているか
